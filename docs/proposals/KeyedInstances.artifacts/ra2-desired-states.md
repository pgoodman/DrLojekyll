  [DS-ADJ-1 — UPHELD (crit F-1, adjudicator-reverified at code + regen).]
  THREADED-PARAM ORDER IS vec25-BEFORE-vec29 (vector-id ASCENDING), NOT
  vec29-before-vec25 as design §C and this ds's first draft drew it. The
  primary-proc params are built by `std::set_intersection(written_by_entry,
  read_by_primary, …, CompareVectors())` (Procedure.cpp:642-644) — output is
  ascending `vec->id` — and BOTH `input_vecs.Create` (:650-652) and the call

  mechanism and are NOT id-sorted — do not use them as the primary-proc
  order model.) No correctness impact (decl/def/call are jointly generated

  MANDATED ADJ-R5 on-landing dump review, so every drawn signature/call
  below is corrected to vec25-then-vec29 and the [STRUCT] "exact slot order
